# Research Report

## Overview

The next-gen RPG toolchain is being designed in a research vault (`rpg-suite-02`). Insights
about the *system* — workflow friction, drift sightings, patterns worth stealing, "wish X
existed" moments, findings from old material — often surface mid-session or mid-workshop,
far from that vault. This skill formats them as a **report** the user drops into the
vault's `inbox/`, where they get mined into atomic notes.

**Core principle: one report = one or more atomic observations, each verbatim-anchored.**
The report is raw material, not a finished note — favor exactness over polish.

## When to use

- A realization about the *system* (not the story) happens mid-play or mid-workshop
- Friction: a workflow, tool, or data format fought you
- A pattern in old material (campaigns, docs, exports) seems worth carrying forward
- A design idea for the next-gen toolchain comes up
- End of a session, when asked "anything system-worthy happen?"

**Not for:** story content, session summaries, campaign canon — those have their own flows.

## How to write the report

1. Copy the structure from `report-template.md` (same folder as this skill).
2. One `## Observation` section per distinct idea — don't blend them. Three small sharp
   observations beat one mushy essay.
3. **Quote verbatim wherever possible** — the user's exact words, the exact error, the
   exact prose that worked. Paraphrase loses the evidence.
4. Fill the context header honestly; "don't know" is a fine value.
5. Suggest `kind/` and `theme/` hints if any fit, but never force them — the vault decides.
6. Name the file `report-YYYY-MM-DD-short-slug.md` and give it to the user as a file or
   copyable block. The user drops it into `rpg-suite-02/inbox/` themselves.

## Why this format (do not flatten)

- **One observation per section** — the vault stores *atomic* notes; blended observations
  have to be untangled by hand later.
- **Verbatim quotes** — the vault has repeatedly found that paraphrase drifts; exact words
  are evidence ("keeps the story from mutating" applies to research too).
- **Hints, not tags** — unconstrained tagging was tried in the campaign system and produced
  hallucinated, useless tags. Hints from a suggested vocabulary are proposals; the vault's
  appraisal step decides.
- **Context header** — a report's value decays fast if nobody can tell where it came from;
  provenance is what makes a claim checkable later.
