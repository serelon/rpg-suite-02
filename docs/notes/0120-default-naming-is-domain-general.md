# The default-name problem is domain-general — generators are first-class namesets

**What it is.** The model's pull toward a tiny default naming vocabulary
([[0013-counter-training-name-the-default]], the "Marcus Chen" problem) is **not specific to
people**. Places, ships, organizations, factions, and handles collapse to the same handful of
defaults just as hard — every starship, station, megacorp, or mystical order drifts toward
*"…Meridian"*, Aurora, Nexus, Helix, Vantage, the Obsidian Something. The counter-default
device — namesets — therefore must cover **all** generated names, not just person-names.

**Where it comes from.** Owner, 2026-06-18 (nameset mining walk): *"kindof tired of everything
being named 'something something meridian'."* A direct field observation of the default-pull in
the non-person domain.

**Why it matters for next-gen.** Resolves the open thread in [[0119-core-namesets-by-demand]]:
the **generator toolkit** (corp/org names, handles/callsigns, ship registries, designations,
place names) is a **first-class, co-equal category** of the core nameset library — not a
lesser cousin and not a separate tool-type. Two consequences:
- **Stock it generously**, alongside person-leaves and era/genre packs (A–C in 0119).
- The non-person cliché is arguably *louder* than the person one: a campaign has far fewer
  place-names than NPCs, so each repeated default ("Meridian") echoes harder. Underserving
  generators is a *more* visible failure, not a niche one.

**Open threads.** Procedural-pattern *mechanics* (random-pattern formats, the lumina handle
stacks, asymptote corp generators) are an implementation flavor that overlaps
[[0087-spreadsheet-semantics-tool-data]] — but that's an authoring detail, not a typing split.
The default place/ship/org vocabulary is worth an explicit **banned/overused list**
(cf. [[0076-self-canonizing-hallucinations]]'s banned-list defense; [[0114-voice-vs-setting-fidelity]]
on default-association errors) so the counter-default has a concrete referent.
