# The provenance graph is the "secret history" — capture lineage as a first-class artifact

**What it is.** From the import brief ([[import-design-brief]]): the routing edges
([[0108-multi-label-relational-routing]]) aren't just for filing — `seed-of`,
`variant-of`, and `crossover-between` together **reconstruct how settings actually
evolved.** Which floating oneshot seeded which canon branch (the "lone consciousness on an
alien world" → Pale Horizon / Parniyan branch); which failed attempts orbit which success
(three failed Sophie-drops pointing at the one that worked); which conversation sits
genuinely between two settings. The brief's sharp line: **folder-routing alone would
discard this as residue.** Lineage is a first-class artifact, not a byproduct.

**Where it comes from.** `../solorpg/imports/IMPORT-DESIGN-BRIEF.md`, "Provenance graph
(the secret history)." The user's own pre-planning.

**Why it matters for next-gen.** The same lineage shape recurs across the whole research:
Tarot Tales settings resurfacing as `solorpg` campaigns (the gen-2→gen-5 lineage in
[[relevant-paths]]), oneshots graduating ([[0065-oneshots-as-spawning-pool]]), branches
and retcons ([[0103-bitemporal-subentry-versioning]]). A provenance/genealogy graph is the
single structure that records *all* of it — "where did this come from, what did it become."
It's the creative-history counterpart to the data-provenance audit trail
([[0057-compiled-context-needs-audit-tooling]], [[0102-catalogue-metadata-shape]]): one
tracks how a *fact* was compiled, the other how an *idea/setting* descended.

**Open threads.** Is this the same graph as the in-play entity/relationship graph, or a
separate meta-graph over campaigns? Granularity — lineage between conversations, between
campaigns, or both? Does it subsume the branch/timeline model ([[0103]]) or sit beside it?
How does it get *maintained* during play, not just reconstructed at import time?

**Verdict.** _(unevaluated.)_
